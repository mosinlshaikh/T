\# T UI/UX Design System



\*\*Project:\*\* T

\*\*Maintainer:\*\* T Technology Research Lab

\*\*Status:\*\* Public Alpha / Research Software



```text

Research only. Not financial advice.

```



\## 1. Purpose



This document defines the UI/UX direction for T.



T should feel like a professional research operating system, not a hype-driven trading application.



The interface should communicate:



\* Research

\* Safety

\* Evidence

\* Risk awareness

\* Human review

\* System clarity

\* Professional discipline



\## 2. Core UI Principle



T should be designed as:



```text

Mission-control interface for financial research.

```



T should not be designed as:



```text

Casino-style trading app.

Signal-selling dashboard.

Profit-hype interface.

```



\## 3. Visual Identity



The preferred visual direction is:



\* Clean

\* Minimal

\* Research-lab style

\* Dashboard-first

\* Metric-driven

\* Risk-aware

\* Professional

\* Calm and serious



Avoid:



\* Overuse of bright green/red

\* Flashy trading slogans

\* Aggressive profit language

\* Buy/sell buttons

\* Confusing charts

\* Overcrowded layouts



\## 4. Dashboard Layout



Recommended dashboard structure:



1\. Header

2\. Research-only safety banner

3\. System status / mission control

4\. Backtest analytics metrics

5\. Equity curve visualization

6\. Trade table

7\. Research safety panel

8\. Hallucination guard status

9\. Documentation links

10\. Footer signature



\## 5. Safety-First UI Rules



Every user-facing interface should make the research-only position clear.



Required safety message:



```text

Research only. Not financial advice.

```



The dashboard should avoid language that sounds like:



\* Direct trading instruction

\* Return assurance

\* Prediction certainty

\* Live execution command

\* Portfolio management advice



Preferred language:



\* Research observation

\* Backtest result

\* Risk note

\* Human review required

\* Historical simulation

\* Evidence summary

\* System status



\## 6. Color Meaning



Recommended color meaning:



| Color Meaning        | Usage                                         |

| -------------------- | --------------------------------------------- |

| Blue / gray          | Research, neutral status, system information  |

| Green                | OK, passed checks, healthy status             |

| Amber                | Caution, review needed, incomplete data       |

| Red                  | Missing item, blocked output, risk warning    |

| Purple / dark accent | Intelligence layer, future AI research module |



Colors should support clarity, not hype.



\## 7. Dashboard Cards



Dashboard cards should be simple and readable.



Recommended cards:



\* System

\* Mode

\* Target release

\* Status

\* Starting balance

\* Ending balance

\* Net PnL

\* Max drawdown

\* Total trades

\* Win rate

\* Profit factor

\* Average return

\* Safety status

\* Human review status



\## 8. Charts



Charts should be used for research clarity.



Recommended charts:



\* Equity curve

\* Drawdown curve

\* Trade PnL distribution

\* Win/loss summary

\* Strategy comparison

\* Risk score trend

\* Data quality trend



Avoid chart clutter. One chart should answer one question.



\## 9. Trade Table UX



Trade table should clearly show:



\* Asset

\* Side

\* Entry price

\* Exit price

\* PnL

\* Return percentage

\* Alpha score

\* Z-score

\* Risk notes if available



Trade table should not include direct buy/sell instructions.



\## 10. Research Safety Panel



The safety panel should show:



\* Research-only status

\* Hallucination guard status

\* Unsafe claim blocking status

\* Human review reminder

\* Disclaimer visibility

\* Output safety state



Example:



```text

Safety Status: Research Mode

Execution: Disabled

Output Guard: Active

Human Review: Required

```



\## 11. Future AI Research UI



Future AI modules should be displayed as research assistants, not autonomous decision makers.



Preferred names:



\* AI Research Report

\* T Research Brain

\* Research Memory

\* Trust Score

\* Risk Explanation Engine

\* Audit Log



Avoid:



\* Auto-profit AI

\* Autonomous trading brain

\* Buy/sell AI

\* Prediction machine



\## 12. Trust Score UX



Future Trust Score should explain why a research output is more or less reliable.



Possible factors:



\* Data quality

\* Sample size

\* Backtest coverage

\* Safety guard status

\* Risk level

\* Missing fields

\* Market volatility

\* Human review status



Example:



```text

Research Trust Score: 72/100

Reason: Backtest exists and safety guard passed, but sample size is limited.

```



\## 13. Multi-Language UX



Future dashboard language support may include:



\* English

\* Hindi

\* Hinglish

\* Spanish

\* Arabic

\* French

\* Portuguese



The default interface language should remain English for global technical users.



Translated text should be short, clear, and safety-reviewed.



\## 14. Accessibility



The UI should be readable and simple.



Recommended:



\* Clear headings

\* Short labels

\* Good spacing

\* High contrast

\* Avoid tiny text

\* Avoid overloaded tables

\* Avoid too many colors

\* Use plain language
