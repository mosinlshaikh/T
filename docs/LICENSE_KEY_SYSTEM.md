# TTRL App License Key System

This system generates TTRL app activation/license keys for client access
control. It is not a Binance API key generator.

The license system is part of the **TTRL AI Trading OS / T Financial
Intelligence OS** architecture created by **MOSIN LIYAKAT SHAIKH** for
T TECHNOLOGY RESEARCH LAB.

## Difference From Binance API Keys

- TTRL license key: generated by the owner/admin backend to activate app/backend
  access.
- Binance API key: created by the user inside Binance and configured separately
  through the secure backend/vault process.

The Android app never creates license keys and never stores Binance credentials.

## Key Format

Example:

`TTRL-XXXX-XXXX-XXXX-XXXX`

The full key is returned only once during admin creation. Persisted records store
the hashed key only.

## License Types

- `OWNER`
- `INTERNAL_TEST`
- `CLIENT_TRIAL`
- `CLIENT_MONTHLY`
- `CLIENT_YEARLY`
- `CLIENT_LIFETIME`

## Statuses

- `ACTIVE`
- `DISABLED`
- `EXPIRED`
- `REVOKED`
- `SUSPENDED`

## Backend Routes

- `POST /admin/licenses/generate`
- `GET /admin/licenses`
- `GET /admin/licenses/{license_id}`
- `POST /admin/licenses/{license_id}/disable`
- `POST /admin/licenses/{license_id}/revoke`
- `POST /admin/licenses/{license_id}/suspend`
- `POST /admin/licenses/{license_id}/activate`
- `POST /license/validate`

Admin routes require `TTRL_ADMIN_TOKEN`. If the token is not configured, admin
generation returns `ADMIN_AUTH_NOT_CONFIGURED`.

## Safety Boundaries

- No Binance secrets are involved.
- No payment integration exists yet.
- No withdrawal permission is supported.
- Live trading remains disabled by default.
- License validation controls TTRL app/backend access only.

## Phase 10 Review Result

The final pre-APK smoke checks verify:

- server-side generation works;
- admin generation is blocked when `TTRL_ADMIN_TOKEN` is not configured;
- the full key is returned only once at creation;
- persisted records store `key_hash`, not the plain license key;
- active, invalid, expired, revoked, and suspended validation states are handled;
- Android source only calls `/license/validate` and never generates keys.
