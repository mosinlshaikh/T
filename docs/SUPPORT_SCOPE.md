\# T Support Scope



\*\*Project:\*\* T

\*\*Maintainer:\*\* T Technology Research Lab

\*\*Status:\*\* Public Alpha / Research Software



```text

Research only. Not financial advice.

```



\## 1. Purpose



This document explains what type of support may be provided for T.



T is open-source public alpha research software. Support may be provided for setup, configuration, dashboard usage, documentation, troubleshooting, and customization guidance.



T is not provided as a managed investment service, trading advisory service, portfolio management service, or live execution service.



\## 2. Supported Assistance



T Technology Research Lab may provide guided support for:



\* Local installation

\* Python environment setup

\* Dependency installation

\* GitHub repository setup

\* Dashboard launch

\* Backtest command usage

\* Documentation walkthrough

\* Basic troubleshooting

\* Dashboard customization guidance

\* Report customization guidance

\* Data integration planning

\* Deployment guidance

\* Training and walkthrough sessions



\## 3. Support Not Included



Support does not include:



\* Financial advice

\* Investment advice

\* Trade recommendations

\* Portfolio management

\* Broker account management

\* Tax advice

\* Legal advice

\* Market prediction assurance

\* Return assurance

\* Live trading operation

\* Emergency trading support

\* 24/7 production SLA unless separately agreed in writing



\## 4. Deployment Support Position



Deployment support means assisted technical guidance.



It may include:



\* Environment preparation

\* Server setup guidance

\* Docker guidance

\* Streamlit dashboard launch help

\* Configuration review

\* Log review

\* Basic troubleshooting

\* Handover documentation



Deployment support does not mean that T Technology Research Lab becomes responsible for a user’s trading decisions, legal compliance, security operations, broker activity, or financial outcomes.



\## 5. User Responsibility



Users are responsible for:



\* Their own data

\* Their own credentials

\* Their own legal compliance

\* Their own server security

\* Their own trading/investment decisions

\* Their own tax and regulatory obligations

\* Reviewing all research output before use



\## 6. Recommended Support Levels



| Support Level       | Description                                                        |

| ------------------- | ------------------------------------------------------------------ |

| Basic Setup         | Local install, dashboard run, walkthrough                          |

| Professional Setup  | Installation, training, dashboard explanation, basic customization |

| Deployment Guidance | VPS/Docker/server guidance and troubleshooting                     |

| Custom Integration  | Data source, report, dashboard, or workflow customization          |

| Monthly Support     | Ongoing technical support within agreed scope                      |

| Enterprise/R\&D      | Custom research and development engagement                         |



\## 7. Communication Requirements



For effective support, users should provide:



\* Operating system

\* Python version

\* Error message

\* Screenshot if useful

\* Command used

\* Relevant log output

\* Repo branch name

\* Expected behavior

\* Actual behavior



Users should never share:



\* API secrets

\* Passwords

\* OTPs

\* Recovery codes

\* Private keys

\* Wallet seed phrases

\* Broker credentials



\## 8. Security Notice



T Technology Research Lab will not ask for wallet seed phrases, private keys, OTPs, recovery codes, or unrestricted access to financial accounts.



If any setup requires credentials, users should configure them directly in their own environment.



\## 9. Official Disclaimer

```text
T is research-only software.

It does not provide financial advice.

It does not provide investment advice.

It does not provide trade recommendations.

Users are responsible for their own decisions and compliance.

```
